# Research Plan

## 当前状态

当前 lexer 已经具备工程可用性：

- 可独立编译和运行
- 具备手写 NFA、DFA 确定化与 DFA 最小化
- 支持最长匹配和结构化诊断
- 可输出 token 序列与自动机统计信息

这意味着后续工作重点不再是“为完整 C-- compiler 补全前端流水线”，而是把 lexer 本身当作研究对象。

## 研究定位

后续目标不是继续服务原 `c--compiler` 主流程，而是围绕词法分析这一主题做：

- 专题实验
- 论文方法复现
- 实现对比
- 可解释性分析
- 性能实验

## 研究方向

1. 正则表达式导数

   用导数法替换或对照 Thompson NFA 构造，比较规则表达能力、实现复杂度和状态规模。

2. TDFA

   研究 Tagged DFA / TDFA 对更复杂词法现象、子匹配信息和扩展诊断的支持能力。

3. 增量词法分析

   面向编辑器场景，探索局部修改后的 token 增量更新，而不是全量重扫。

4. 形式化验证思想 / property-based testing

   引入更系统的自动化测试方法，验证 lexer 在等价输入变换、随机输入和错误恢复场景下的性质。

5. DFA 可视化与最小化解释

   输出 NFA / DFA / 最小 DFA 图和分组细化过程，服务教学展示和研究报告。

6. 高性能扫描与字符分类表

   引入字符分类表、压缩转移表、cache-friendly 布局或 SIMD 友好策略，评估性能提升空间。

## 扩展原则

- 保持核心 API 稳定
- 保持词法模块对 parser / AST / IR 零依赖
- 新研究实现优先并行存在，不直接覆盖稳定实现
- 先保留可对照基线，再引入实验性优化
